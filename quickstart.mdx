## Prerequisites

Before you begin, make sure you have:

1. **Harness CLI installed** — see [Installation](/installation)
2. **At least one agent installed** — run `harness list` to verify

<Info>
If you don't have an agent installed yet, head to the [agent installation guide](/installation#system-requirements) to set one up.
</Info>

## Run Your First Task

<Steps>
  <Step title="Basic usage">
    Run an agent with a simple prompt:

    ```bash
    harness run --agent claude --prompt "explain this codebase"
    ```

    Harness will:
    - Spawn the Claude Code agent as a subprocess
    - Stream its output as unified NDJSON events
    - Display the results to your terminal
  </Step>

  <Step title="Understanding the output">
    By default, Harness outputs **NDJSON** (newline-delimited JSON) — one event per line:

    ```json
    {"type":"session_start","session_id":"session-1234","agent":"Claude Code","model":"claude-3-7-sonnet-20250219","timestamp_ms":1709548800000}
    {"type":"text_delta","text":"Let me analyze","timestamp_ms":1709548801000}
    {"type":"text_delta","text":" this codebase for you.","timestamp_ms":1709548801100}
    {"type":"tool_start","call_id":"call_abc123","tool_name":"read_file","input":{"path":"README.md"},"timestamp_ms":1709548802000}
    {"type":"tool_end","call_id":"call_abc123","tool_name":"read_file","success":true,"timestamp_ms":1709548803000}
    {"type":"usage_delta","usage":{"input_tokens":1500,"output_tokens":250,"cost_usd":0.015},"timestamp_ms":1709548804000}
    {"type":"result","success":true,"text":"This codebase implements...","session_id":"session-1234","duration_ms":4200,"total_cost_usd":0.015,"timestamp_ms":1709548805000}
    ```

    <Note>
    Each event has a `type` field indicating the event kind. See [Event Stream](/concepts/event-stream) for details on all event types.
    </Note>
  </Step>

  <Step title="Use different output formats">
    Change the output format with the `--output` flag:

    <CodeGroup>

    ```bash Text output
    harness run --agent claude --prompt "hello" --output text
    ```

    ```bash JSON output
    harness run --agent claude --prompt "hello" --output json
    ```

    ```bash Markdown output
    harness run --agent claude --prompt "hello" --output markdown
    ```

    </CodeGroup>

    - **`text`** — Only prints the final assistant message (clean, minimal)
    - **`json`** — Single JSON object with the final result
    - **`stream-json`** — NDJSON events as they happen (default)
    - **`markdown`** — Human-readable transcript with headings and code blocks

    <Tip>
    Use `--output text` when piping Harness output to other tools, and `--output markdown` for human-readable logs.
    </Tip>
  </Step>
</Steps>

## Common Usage Patterns

<CodeGroup>

```bash Specify a model
harness run --agent claude --model opus --prompt "fix the bug"
```

```bash Pipe prompt from stdin
echo "explain this codebase" | harness run --agent claude
```

```bash Use a model alias
harness run --agent claude --model sonnet --prompt "refactor this function"
```

```bash Dry-run to see the command
harness run --agent claude --model sonnet --prompt "hello" --dry-run
```

</CodeGroup>

## Understanding the Event Stream

Harness translates each agent's native output into a unified event stream with 8 event types:

| Event Type | Description |
|------------|-------------|
| `SessionStart` | Agent session initialized |
| `TextDelta` | Streaming text chunk from the assistant |
| `Message` | Complete message (user, assistant, or system) |
| `ToolStart` | Agent is invoking a tool |
| `ToolEnd` | Tool invocation completed |
| `UsageDelta` | Incremental token usage and cost update |
| `Result` | Final result of the agent run |
| `Error` | An error occurred |

<Info>
Regardless of which agent backend you use (Claude, OpenCode, Codex, or Cursor), you always get the same event structure.
</Info>

## Example: Text Output Mode

When you use `--output text`, you only see the final assistant message:

```bash
harness run --agent claude --prompt "what is 2+2" --output text
```

Output:

```text
2 + 2 equals 4.
```

The cost summary is printed to stderr:

```text
Total: 150 in / 10 out, $0.002, 1.2s
```

## Example: JSON Output Mode

With `--output json`, you get a structured result object:

```bash
harness run --agent claude --prompt "what is 2+2" --output json
```

Output:

```json
{
  "type": "result",
  "success": true,
  "result": "2 + 2 equals 4.",
  "session_id": "session-1709548800000-12345"
}
```

## Auto-Detection

If you only have one agent installed, you can omit the `--agent` flag:

```bash
harness run --prompt "explain this file"
```

Harness will automatically detect and use the available agent.

<Warning>
If multiple agents are installed, you must specify which one to use with `--agent`, or set a default in your [configuration file](/guides/configuration).
</Warning>
